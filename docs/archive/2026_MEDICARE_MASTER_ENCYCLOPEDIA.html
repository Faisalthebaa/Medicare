The Strategic Goal

Stopping Information Blindness

In 2026, healthcare data is a mess of disconnected files. Our mission is to unify these files into a single Command Center. We aim to:

  • Identify Leakage: Find where the $1.2M is being lost.
  • Predict Risk: Stop patients from getting sick after discharge.

Phase 1: The Vault SQL Engineering

Architecting for Speed & Integrity

Why do we use SQL instead of Excel or raw CSV files?

  • Lightning Speed: SQL queries run up to 75x faster on large datasets compared to legacy spreadsheets.
  • Data Integrity: We use Primary & Foreign Keys to ensure no medical record is ever duplicated or lost.
  • Automation: Materialized views run complex calculations in the background automatically.

Phase 1 is about building the "Brain" of the project. We design a PostgreSQL 16 Data Warehouse to act as the single source of truth.

SQL ARCHITECTURE: > CREATE SCHEMA production_medicare;
> CREATE TABLE fact_admissions (PK_id, FK_patient...);
> CREATE INDEX idx_readmit ON admissions(readmitted_30d);
The Data Analyst Advantage: We use 3rd Normal Form (3NF) to eliminate data redundancy, making our analytics platform robust and instantly scalable.

Phase 2: The Vibe Check Python EDA

Data is never clean. We use Python & Pandas to perform a surgical "Vibe Check"—cleaning 30,000 rows of errors before analysis.

ANALYTICS PIPELINE: [1] df = pd.read_sql('SELECT * FROM admissions')
[2] df.fillna({'outstanding_balance': 0}, inplace=True)
[3] df['is_elderly'] = df['age'] > 65
The Data Analyst Advantage: We use Vectorized Operations to clean data in seconds. We find the "Red Flags"—like billing dates that happen before a patient was even admitted.
